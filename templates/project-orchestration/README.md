# Project Orchestration — {PROJECT_NAME}

This folder is the **project management system** for {PROJECT_NAME}.
It uses the [Project Management System pattern](../../patterns/orchestration/project-management-system.md)
from claude-patterns.

---

## Structure

```
project-orchestration/
├── TEAM-STATE.md          ← Shared brain — auto-updated by /pulse
├── KANBAN.md              ← Task board — auto-generated by /pulse
├── TECH-DEBT.md           ← Debt register — updated by @tech-lead
├── README.md              ← This file
│
├── tasks/                 ← Active tasks (YAML frontmatter)
├── completed-tasks/       ← Completed tasks (archive, immutable)
└── _archive/              ← Historical docs, analyses, old files
```

---

## Daily Workflow

```bash
# Morning: check state without spawning agents (~$0)
/pm-status

# Full sync: run both agents, update TEAM-STATE.md (~$0.10)
/pulse

# Before sprint planning
/sprint
```

---

## Task Lifecycle

```
New task → tasks/{TS-XXX}.md (status: planned)
         → status: ready (dependencies met)
         → status: in-progress (work started)
         → status: done
         → move to completed-tasks/{TS-XXX}.md
```

---

## Task YAML Schema

Minimum required:
```yaml
---
id: TS-XXX
title: 'Task title'
status: planned|ready|in-progress|blocked|done|deferred
priority: P0|P1|P2|P3
story_points: 5
created_date: YYYY-MM-DD
updated_date: YYYY-MM-DD
assignee: '@agent-or-person'
labels: [area, type]
---
```

Extended (enables PM system intelligence):
```yaml
due_date: YYYY-MM-DD            # overdue detection
mobile_impact: none|low|medium|high  # UX risk flag
tech_debt: none|minor|major     # debt aggregation
dependencies: [TS-AAA]          # upstream blockers
blocks: [TS-BBB]               # reverse dependencies
story_id: US-XXX               # business traceability
```

## Available Skills

| Skill | Purpose | Cost |
|-------|---------|------|
| `/pm-status` | Quick state read (TEAM-STATE.md only) | ~$0 |
| `/pulse` | Full team sync (both agents) | ~$0.10 |
| `/task-tidy` | Housekeeping: move done tasks, fix fields, validate YAML | ~$0.03 |
| `/task-health` | Deep task audit (broken deps, stale, orphaned) | ~$0.05 |
| `/tech-debt` | Debt analysis + TECH-DEBT.md update | ~$0.05 |
| `/sprint` | Interactive sprint planning | ~$0.20 |

## Advisory Agents

| Agent | When to call | What they analyze |
|-------|-------------|-------------------|
| `@tech-lead` | Technical questions, blocked deps, debt | tasks/, git log, BUSINESS_RULES.yaml |
| `@product-owner` | Business value, mobile UX, priorities | tasks/, product docs, validation evidence |

---

## Conventions

- **Always update `updated_date`** when editing a task
- **Move done tasks immediately** to `completed-tasks/`
- **Old files go to `_archive/`** — keep root clean
- **Don't edit TEAM-STATE.md manually** — use `/pulse`
- **Don't edit KANBAN.md manually** — auto-generated
